方法调用模式

函数调用模式

以此模式调用函数时，this被绑定到全局对象

构造器调用模式

如果在一个函数前面带上new来调用，那么背地里将会创建一个连接到该函数的prototype成员的新对象，同时this会被绑定到那个新对象上。

Apply调用模式

作用域

JavaScript 有函数作用域，定义在函数中的参数与变量在函数外部不可见
而在一个函数内部的任何位置定义的变量，在该函数内部任何地方都可见

闭包

函数可以访问它被创建时所处的上下文环境

模块

模块模式的一般形式是：一个定义了私有变量和函数的函数；利用闭包创建可以访问私有变量和函数的特权函数（拥有使用或修改私有状态的能力）；最后返回了这个特权函数，或者把它们保存到一个用来产生序列号的对象。

级联

方法返回this，可以启用级联

基于原型继承

摒弃类，专注于对象。
基于原型的继承相比于基于类的继承在概念上更简单：一个新对象可以继承一个旧对象的属性

利用 Object.create 方法创建新对象（第三章 原型），构造一个已经存在对象的新实例
原型连接在更新时不起作用（即不会改变原对象的属性值）
原型连接只有在检索值的时候才被用到

部件

数组

类数组（array-like），实质是对象，并且对象可接受任何字符串作为属性名

删除

  • delete

    delete numbers[2] 会在数组中留下一个空洞，排在被删除元素之后的元素保留着最初属性

  • splice

    splice(index, num)，删除元素并可以替换为其他元素。
    index 数组中要删除元素需要
    num 要删除元素个数
    被删除属性后面的每个属性被移除，并且以一个新的键值重新插入

判断是否是数组

typeof 运算符报告数组的类型是 ‘object’

  • 方法一

    缺点： 识别从不同的窗口（window）或帧（frame）里构造的数组时会失败

  • 方法二

方法

  • Array.prototye 可以被扩充
  • 数组是对象，可以直接给单独的数组添加方法
  • 使用 Object.create 方法用在数组没有意义，因为它产生一个对象，而不是一个数组。产生的对象将继承这个数组的值和方法，但它没有那个特殊的 length 属性。

    即：继承了 Object.prototype, 未继承 Array.prototype

毒瘤

全局变量

三种方式声明全局变量

  • 在任何函数之外放置一个 var 语句

  • 直接给全局对象添加一个属性

  • 直接使用未经声明的变量，称为隐式的全局变量

作用域

JavaScript 有块语法，但是没有提供块级作用域：代码块中声明的变量在包含此代码块的函数的任何位置都是课件。

推荐：在每个函数的开头部分申明所有的变量

自动插入分号

JavaScript 自动修复机制：通过自动插入分号来修正有缺损的程序。

自动插入分号，导致返回 undefined
把 { 放在上一行的末尾

保留字

不能被用来命名变量或参数。当保留字被用做对象字面量的键值时，必须加引号括起来。

不能被用在点表示法中，使用括号表示法。

typeof

  • typeof null 返回 ‘object’

parseInt

  • 遇到非数字时会停止解析

  • 如果该字符串第一个字符是0， 会基于八进制求值。parseInt可以接受一个基数作为参数

浮点数

  • 二进制的浮点数不能正确处理十进制的小数，即，0.1 + 0.2 不等于 0.3
  • 浮点数中的整数运算时精准的。

    举例：美元可以通过乘以 100 全部转成美分，准确将美分相加。和再除以 100 转回美元

NaN

  • NaN 表示不是一个数字

  • typeof 不能辨别数字和 NaN

  • NaN 不等同于它自己

  • JavaScript 提供 isNaN 函数，辨别数字与NaN

  • isFinite 函数判断一个值是否可用做数字，筛选掉NaN 和 Infinity；但是它会试图把它的运算数转化为一个数字

伪数组

  • JavaScript 没有真正的数组
  • typeof 不能辨别数组和对象

假值

表A-1：JavaScript 的众多假值
|值|类型|
|—|—|
|0|Number|
|NaN(非数字)|Number|
|’’(空字符串)|String|
|false|Boolean|
|null|Object|
|undefined|Undefined|

这些值全部都等同于假

undefined是缺失的成员属性的值

糟粕

==

  • === 与 !== ：判断两个运算数类型一致且拥有相同的值
  • == 与 != ：当两个值是不同的类型，它们会去强制转换值的类型；缺乏传递性

with

本意用来快捷地访问对象属性；但是结果不可预料，避免使用；影响 JavaScript 处理器的速度，因为它阻断了变量名的词法作用域绑定

eval

  • 使代码难以阅读
  • 性能降低
  • 减弱安全性

continue

  • 降低性能

switch 穿越

  • 危险

  • 危险
