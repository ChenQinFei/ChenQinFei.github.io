盒模型

margin叠加

外边距叠加是一个相当简单的概念。 但是，在实践中对网页进行布局时， 它会造成许多混淆。 简单的说， 当两个或更多个垂直边距相遇时， 它们将形成一个外边距。这个外边距的高度等于两个发生叠加的外边距的高度中的较大者。但是注意只有普通文档流中块框的垂直外边距才会发生外边距叠加。 行内框、 浮动框或绝对定位框之间的外边距不会叠加。

一般来说， 垂直外边距叠加有三种情况：

  • 元素自身叠加 当元素没有内容（即空元素）、内边距、边框时， 它的上下边距就相遇了， 即会产生叠加（垂直方向）。 当为元素添加内容、 内边距、 边框任何一项， 就会取消叠加。
  • 相邻元素叠加 相邻的两个元素， 如果它们的上下边距相遇，即会产生叠加。
  • 包含（父子）元素叠加 包含元素的外边距隔着 父元素的内边距和边框， 当这两项都不存在的时候， 父子元素垂直外边距相邻， 产生叠加。 添加任何一项即会取消叠加。

block、inline、inline-block、table-cell

block

  • 独占一行
  • block元素可以设置width,height属性。块级元素即使设置了宽度,仍然是独占一行。
  • 可以设置margin、padding属性

inline

行内非替换元素（反例：img）

  • inline元素不会独占一行，多个相邻的行内元素会排列在同一行里。其宽度随元素的内容而变化
  • inline元素设置width,height属性无效
  • 垂直方向设置margin无效（由于外边距是透明的，无任何视觉效果）
  • 行内框高度受line-height影响。
  • 行内框的边框边界由font-size控制，包围内容区。

其他

img 即是inline，又是replace元素。类似inline-block表现

CSS 结构和层叠

优先级

选择器的特殊性由选择器本身的组件确定。特殊性值表述为4个部分，如：0,0,0,0.一个选择器的具体特殊性如下确定：

  • 给定的各个ID属性值，加 0,1,0,0
  • 给定的各个类属性值、属性选择或伪类，加0,0,1,0
  • 给定的各个元素和伪元素，加 0,0,0,1
  • 结合符和通配选择器对特殊性没有任何贡献
  • 内联样式特殊性，加 1,0,0,0

重要性 !important

  1. 标志为！important的声明没有特殊的特殊属性

  2. 所有！important声明会分组在一起，其冲突会在重要声明内部解决

  3. 重要声明优先级总是高于非重要声明

继承

  • 继承向下传播，特例：body元素的背景样式可以传递到html元素，可以定义其画布
  • 存在属性无法继承：border，框模型属性(padding, margin, background, border)
  • 继承属性无特殊性，即0特殊性强于无特殊性

++ 操作节点 ++

  • var returnedNode = someNode.appendNode()

    插入后成为最后一个子节点
    向childNodes列表末尾添加一个节点，返回新增节点

  • var returnedNode = someNode.insertBefore(newNode, someNode.firstNode())

    插入后成为第一个子节点
    被插入节点变成参照接待前一个同胞节点（previousSibling），同时被方法返回

  • var returnedNode = someNode.replaceChild(newNode, someNode.firstChild)

    替换第一个子节点
    要替换的节点被返回并从文档树中移除

  • var returnedNode = someNode.removeChild(someNode.firstChild)

    移除第一个子节点

  • var deepList = myList.cloneNode(true)

  • var shallowList = myList.cloneNode(false)

    参数为true，执行深复制，复制节点及其整个子节点数；浅复制，只复制节点本身

  • normalize()

    处理文档树中的文本节点

事件

阻止事件冒泡

  1. event.stopPropogation
  2. event.cancelBubul = true
  3. 直接删除目标元素。目标元素在文档中是事件冒泡的前提。
